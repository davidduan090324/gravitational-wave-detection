# 引力波检测训练流程摘要

本文围绕当前项目中的引力波检测训练代码，总结其数据处理流程、模型结构选择和实验组织方法。项目针对 G2Net 数据集中的三通道时序信号开展二分类任务，目标是判断输入样本中是否存在引力波事件。每个样本由三个探测器的 4096 点原始波形组成。考虑到引力波信号极其微弱，且常常淹没在复杂噪声中，项目没有直接把原始时序送入分类器，而是先将其转换为更适合深度模型提取模式的时频表示。具体而言，代码使用 Constant-Q Transform 对每个探测器通道分别进行变换，再将频谱幅值映射到对数 dB 尺度，最后做单样本归一化并堆叠成三通道频谱图。这一过程保留了信号在时间和频率两个维度上的演化规律，特别有利于表达引力波典型的 chirp 结构。

在模型设计上，当前代码已经统一为一个多模型框架，通过 `--model` 参数即可切换不同架构。默认模型为 EfficientNet，它利用预训练卷积骨干和轻量分类头，在精度与训练速度之间取得较好平衡。ResNet 分支提供了经典残差结构作为对照方案，适合做稳定性实验和结构比较。Transformer 分支则采用 Vision Transformer 思路，把 CQT 频谱图切分为多个 patch，利用自注意力机制建立跨时间、跨频率以及跨探测器的全局依赖关系。从理论上看，这种结构能够提升模型表达上限，尤其适合学习复杂的长程关联。除此之外，项目还保留了一个 SimpleCNN 作为轻量级基线，用于快速验证数据流程或在依赖不足时替代复杂模型。

训练阶段使用 `BCEWithLogitsLoss` 作为二分类目标函数，配合 `Adam` 优化器和余弦退火学习率调度器完成参数更新。代码同时加入了梯度裁剪，以提升深层网络尤其是 Transformer 分支训练时的稳定性。训练集启用轻量增强，包括时间轴翻转和幅值缩放；验证集则保持固定处理流程，以保证评估结果可比较。评价指标方面，项目不仅记录训练和验证损失，也持续统计 Accuracy 与 ROC-AUC，并在训练结束后生成 Precision、Recall、F1、混淆矩阵和 ROC 曲线等结果文件。所有实验输出都会被保存到带时间戳和实验名称的 `results/` 目录中，包括配置、日志、最佳模型权重和可视化图像；同时还支持通过 Weights & Biases 做在线实验追踪。

整体而言，这套训练系统已经形成了一条完整、清晰且可复现实验链路：从原始多探测器时序信号出发，通过 CQT 时频特征构建，把引力波识别问题转化为频谱图上的视觉分类任务，再借助 EfficientNet、ResNet、Transformer 等模型进行统一训练与比较。这种设计既符合当前代码的实际实现逻辑，也为后续进一步提高模型性能、研究不同架构的优劣和扩展更复杂的物理先验提供了坚实基础。
